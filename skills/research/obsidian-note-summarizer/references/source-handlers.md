# 不同内容来源的处理方式

## 网页文章

**处理方式**: 使用 web 工具抓取 → 提取正文 → 结构化摘要
**注意**:
- 遇到反爬/登录墙时，请用户复制粘贴内容
- 保留原始 URL 作为 source 字段
- 标题中的特殊字符需要转义用于文件名

## PDF 文档

**处理方式**: 提取文本 → 去除页眉页脚 → 结构化摘要
**工具**: pymupdf / marker-pdf
**注意**:
- 扫描版 PDF 需要 OCR
- 学术论文保留 DOI 和作者信息

## 微信公众号文章

**处理方式**: 通常有反爬，优先让用户复制粘贴
**备选**: 微信"浮窗"功能配合浏览器插件提取
**注意**:
- 微信文章 URL 有时效性，务必保存全文到笔记
- 排版中的图片可用描述替代

## 会议录音/语音

**处理方式**: 语音转文字 → 提取关键信息 → 会议记录模板
**注意**:
- 区分不同发言人的内容（如有）
- 重点提取决策和行动项

## 用户口述/自由输入

**处理方式**: 直接结构化 → 选择合适模板
**判断**:
- 包含会议要素（参会人、议题）→ 会议记录
- 是观点/知识 → 文章摘要
- 是反思/复盘 → 周期总结
