# v6.0 升级指南 — 从 v5.1 迁移到 v6.0

> 本文档帮助现有 pm-idea-to-mvp v5.1 项目平滑迁移到 v6.0 Loop Engineering 架构

## 1. 变更概览

### 1.1 核心变化

| 维度 | v5.1 | v6.0 |
|------|------|------|
| **架构模式** | 线性流水线 | 递归循环（Why Loop + How Loop） |
| **人工卡点** | G1, G2, G3 全部人工 | 仅 Ship 人工（默认） |
| **决策处理** | 所有决策等权 | Risk-based 分级（low/medium/high） |
| **失败处理** | 阻塞等人工 | Inner loop retry + backtrack |
| **进度追踪** | 手动更新 gates.json | 自动 PROGRESS.md |
| **目标验证** | 人工检查 | goal-check.py 自动验证 |
| **自我改进** | Retro 文档 | skill_patch 自动应用 |

### 1.2 新增文件

| 文件 | 用途 | 必需？ |
|------|------|--------|
| `harness-rules.yaml` | 决策风险分级和自动化规则 | ✅ 必需 |
| `PROGRESS.md` | 项目进度追踪（自动生成） | ✅ 必需 |
| `goals/goals.yaml` | 各阶段目标定义 | ✅ 必需 |
| `references/loop-engineering.md` | 核心理念文档 | 可选 |
| `references/runtime-kanban-v6.0.md` | 运行时详解 | 可选 |
| `references/v6.0-upgrade.md` | 本文档 | 可选 |

### 1.3 修改文件

| 文件 | 变化 |
|------|------|
| `gates.template.json` | 新增 `version`, `mode`, `checkpoint` 字段 |
| `workflow_state.template.yaml` | 新增 `inner_loop_state` 块 |
| `CHANGELOG.md` | 新增 v6.0.0 条目 |

### 1.4 删除文件

无。v6.0 完全向后兼容 v5.1 的所有文件。

## 2. 迁移步骤

### 2.1 新项目（推荐）

新项目直接使用 v6.0 模板，无需迁移：

```bash
# 1. 创建项目目录
mkdir projects/pm-my-idea
cd projects/pm-my-idea

# 2. 复制 v6.0 模板
cp ~/.hermes/skills/pm-idea-to-mvp/assets/gates.template.json gates.json
cp ~/.hermes/skills/pm-idea-to-mvp/assets/harness-rules.template.yaml harness-rules.yaml
cp ~/.hermes/skills/pm-idea-to-mvp/assets/workflow_state.template.yaml workflow_state.yaml
cp ~/.hermes/skills/pm-idea-to-mvp/assets/progress.template.md PROGRESS.md

# 3. 创建 goals 目录
mkdir -p goals
cp ~/.hermes/skills/pm-idea-to-mvp/assets/goal.template.yaml goals/goals.yaml

# 4. 初始化项目
echo "# My Idea" > 00-brief.md
# ... 继续正常流程
```

### 2.2 现有 v5.1 项目迁移

#### 步骤 1：备份现有项目

```bash
cd projects/pm-my-idea
cp -r . ../pm-my-idea-v5.1-backup
```

#### 步骤 2：更新 gates.json

```bash
# 备份旧版本
cp gates.json gates.json.v5.1

# 下载 v6.0 模板
cp ~/.hermes/skills/pm-idea-to-mvp/assets/gates.template.json gates.json
```

手动编辑 `gates.json`，将已有阶段的状态从备份中恢复：

```json
{
  "version": "6.0.0",
  "mode": "loop",
  "stages": {
    "align": { "status": "passed", "gate": "G1", "checkpoint": "auto" },
    "research": { "status": "passed", "checkpoint": "auto" },
    "analysis": { "status": "running", "checkpoint": "auto" },
    "spec": { "status": "pending", "gate": "G2", "checkpoint": "auto" },
    "mvp": { "status": "pending", "gate": "G3", "checkpoint": "auto", "inner_loop": { "max_iterations": 3, "feedback_signals": ["test", "build", "lint"] } },
    "ship": { "status": "pending", "checkpoint": "human" },
    "operate": { "status": "pending", "checkpoint": "auto" },
    "grow": { "status": "pending", "checkpoint": "auto" },
    "retro": { "status": "pending", "checkpoint": "auto" }
  }
}
```

#### 步骤 3：创建 harness-rules.yaml

```bash
cp ~/.hermes/skills/pm-idea-to-mvp/assets/harness-rules.template.yaml harness-rules.yaml
```

编辑 `harness-rules.yaml`，填入项目信息：

```yaml
project:
  slug: 'my-idea'
  tech_stack: python  # 或 node, mixed
  language: zh-CN

runtime:
  test_cmd: 'pytest -q --tb=short'
  build_cmd: 'python -m py_compile **/*.py'
  lint_cmd: 'python -m flake8 --max-line-length=120'
  workdir: '04-mvp'
```

#### 步骤 4：创建 goals/goals.yaml

```bash
mkdir -p goals
cp ~/.hermes/skills/pm-idea-to-mvp/assets/goal.template.yaml goals/goals.yaml
```

根据项目实际情况调整 goal 定义。例如，如果项目不用 pytest，可以修改 `M2`：

```yaml
  mvp:
    - id: M2
      description: 'Tests pass'
      type: command_pass
      command: 'npm test'  # 改为 npm test
      workdir: '04-mvp'
      optional: true
```

#### 步骤 5：创建 PROGRESS.md

```bash
cp ~/.hermes/skills/pm-idea-to-mvp/assets/progress.template.md PROGRESS.md
```

编辑 `PROGRESS.md`，填入当前状态：

```markdown
## 元信息
| 项 | 值 |
|----|----|
| Slug | my-idea |
| 创建时间 | 2026-06-01 10:00 |
| 最后更新 | 2026-06-12 15:00 |
| 当前阶段 | analysis |
| 内循环迭代 | 0/3 |

## 阶段进度
- [x] Stage 0: Brief
- [x] Stage 1: Align (G1)
- [ ] Stage 2: Research  ← current
- [ ] Stage 3: Analysis + C4
...
```

#### 步骤 6：更新 workflow_state.yaml

```bash
# 备份旧版本
cp workflow_state.yaml workflow_state.yaml.v5.1

# 下载 v6.0 模板
cp ~/.hermes/skills/pm-idea-to-mvp/assets/workflow_state.template.yaml workflow_state.yaml
```

手动合并旧状态到新格式：

```yaml
project: 'my-idea'
version: '6.0.0'
mode: loop
current_phase: analysis
target_phase: null
gates:
  g1_align: passed
  g2_spec: pending
  g3_ship: pending
acceptance:
  functional: pending
  ux: pending
  ops: pending
artifacts:
  ssot_status: docs/design/08-CURRENT-STATUS.md
  decisions: docs/DECISIONS.md
  design_tokens: docs/DESIGN.md
  harness_rules: harness-rules.yaml
  progress: PROGRESS.md
  goals: goals/goals.yaml
checkpoints: []
blockers: []
inner_loop_state:
  current_iteration: 0
  max_iterations: 3
  feedback_signals: []
  last_signal: null
  backtracking_log: []
  status: idle
```

#### 步骤 7：验证迁移

```bash
# 1. 验证 gates.json 语法
python -c "import json; json.load(open('gates.json'))"

# 2. 验证 harness-rules.yaml 语法
python -c "import yaml; yaml.safe_load(open('harness-rules.yaml'))"

# 3. 验证 goals/goals.yaml 语法
python -c "import yaml; yaml.safe_load(open('goals/goals.yaml'))"

# 4. 运行 goal check（应该失败，因为还没完成当前阶段）
goal-check.py analysis

# 5. 检查 PROGRESS.md 格式
cat PROGRESS.md
```

#### 步骤 8：继续项目

```bash
# 告诉 Hermes 继续项目
hermes "继续 pm-my-idea"
```

Hermes 会自动检测 v6.0 格式并加载对应 profile。

## 3. 向后兼容性

### 3.1 v5.1 文件仍可读取

v6.0 的 orchestrator 可以读取 v5.1 的 `gates.json`，但会提示升级：

```
⚠️ 检测到 v5.1 格式的 gates.json。建议升级到 v6.0 以启用 inner loop 和 risk-based 决策。
运行 `hermes upgrade-pm-pipeline` 自动迁移。
```

### 3.2 退化为 v5.1 行为

如果不想使用 v6.0 的新特性，可以在 `harness-rules.yaml` 中设置所有决策为高风险：

```yaml
decisions:
  research_supplement:
    risk: high
    action: human_checkpoint
  tech_choice:
    risk: high
    action: human_checkpoint
  architecture_change:
    risk: high
    action: human_checkpoint
  refactor:
    risk: high
    action: human_checkpoint
  deploy:
    risk: high
    action: human_checkpoint
  data_migration:
    risk: high
    action: human_checkpoint
  scope_change:
    risk: high
    action: human_checkpoint

human_checkpoints:
  - align
  - spec
  - ship
```

这样 v6.0 的行为就退化为 v5.1：所有决策都需要人工批准。

### 3.3 混合模式

可以只对特定阶段使用 v6.0 特性：

```yaml
# 只在 MVP 阶段使用 inner loop，其他阶段保持 v5.1 行为
human_checkpoints:
  - align
  - spec
  - ship

inner_loop:
  max_iterations: 3  # 只在 MVP 阶段启用
```

## 5. 常见问题

### 5.1 迁移后 Hermes 报错 "unknown field: checkpoint"

**原因**：`gates.json` 格式未更新。  
**解决**：重新复制 `gates.template.json` 并手动合并状态。

### 5.2 Goal check 总是失败

**原因**：`goals/goals.yaml` 中的 `target` 路径错误。  
**解决**：检查路径是否相对于项目根目录。例如 `04-mvp/README.md` 而不是 `./04-mvp/README.md`。

### 5.3 Inner loop 不触发

**原因**：`workflow_state.yaml` 缺少 `inner_loop_state` 块。  
**解决**：重新复制 `workflow_state.template.yaml` 并合并状态。

### 5.4 想回退到 v5.1

**解决**：参考 3.2 节，将所有决策设为 `risk: high`。
