# Obsidian 笔记深化与聚合

对 Obsidian 知识库进行**深度处理**：单篇笔记深化思考、多篇笔记自动聚合、周期性报告生成、内容地图（MOC）自动维护。

## 什么时候触发

| 用户说的话 | 触发操作 |
|-----------|----------|
| "深化这篇笔记" / "deepen" | 对指定笔记生成批判性问题 |
| "总结一下这个月的笔记" / "月报" | 按月聚合所有笔记生成月报 |
| "生成周报" / "weekly" | 按周聚合所有笔记生成周报 |
| "把关于 XX 的笔记汇总一下" | 按标签/关键词聚合，生成主题复盘 |
| "生成目录" / "MOC" / "内容地图" | 自动扫描文件夹生成导航页 |

## Vault 路径

先加载 `obsidian` skill 获取 vault 路径约定。

路径来自 `OBSIDIAN_VAULT_PATH` 环境变量，或者默认 `~/Documents/Obsidian Vault`。

**文件工具不展开 shell 变量**。调用 `read_file`、`write_file`、`patch` 前必须先解析为绝对路径。

## 核心操作

### 操作 1: 单篇笔记深化（Deepen）

**目标**：对已有笔记生成四类批判性问题，推动思考深化。

**流程**：

1. 读取目标笔记内容
2. 分析笔记的：核心论点、隐含假设、缺失视角、可行动的下一步
3. 在笔记末尾追加一个 `## 🤔 深化提问` 章节，包含四类问题：
   - **批判性问题**：这个观点有什么漏洞？什么情况下不成立？
   - **连接性问题**：和 vault 中哪些已有笔记相关？能否建立 `[[wikilink]]`？
   - **行动性问题**：基于这个笔记，下一步应该做什么？
   - **扩展性问题**：如果深入下去，还需要了解什么？
4. 如果发现了相关的已有笔记，自动在笔记中插入 `[[wikilink]]`

**深化提问模板**（追加到笔记末尾）：

```markdown
---

## 🤔 深化提问（YYYY-MM-DD）

### 🔴 批判性
- 这个观点的隐含假设是什么？如果假设不成立会怎样？
- 有什么反例可以挑战这个结论？

### 🔗 连接性
- [[相关笔记1]] — 这篇笔记的 XX 观点和那篇的 YY 有什么关联？
- [[相关笔记2]] — 两者是否有矛盾？如何调和？

### 🎯 行动性
- [ ] 验证 XX 观点在实际场景中的适用性
- [ ] 和 @某人 讨论 YY 问题

### 🔭 扩展性
- 进一步了解 ZZ 领域的研究
- 对比方案 A 和方案 B 的优劣
```

**注意事项**：
- 每次深化追加**新日期**的章节，不覆盖旧内容 —— 可以看到思考演进
- 使用 `search_files` 在 vault 中寻找相关笔记（按标签、关键词）
- 只链接**确实存在**的笔记

### 操作 2: 周报生成

**目标**：自动聚合本周所有笔记，生成结构化周报。

**流程**：

1. 确定本周日期范围（周一到周日）
2. 扫描 vault 中本周创建/修改的所有 `.md` 文件
3. 读取每篇笔记的 frontmatter 和正文前 300 字
4. 按类别聚合：
   - 完成的任务
   - 参与的会议
   - 阅读的文章/资料
   - 做出的决策
   - 产生的想法
5. 生成周报保存到 `周报/YYYY-WNN.md`

**周报输出模板**：

```markdown
---
type: weekly-review
week: 2026-W21
date_from: 2026-05-19
date_to: 2026-05-25
generated: 2026-05-25
---

# 周报 — 2026年第21周（05/19 - 05/25）

## 📊 本周概览
- 📝 新建笔记 **N** 篇
- ✅ 完成任务 **N** 项
- 📋 参加会议 **N** 场
- 📖 阅读资料 **N** 篇

## 🎯 本周完成
| 任务 | 完成日期 | 相关笔记 |
|------|----------|----------|
| ... | ... | [[笔记]] |

## 📋 会议摘要
### [[会议名称]]（日期）
- 决策：...
- 行动项：...

## 📖 阅读 & 学习
### [[文章标题]]
> 一句话总结：...

## 💡 关键决策
| 决策 | 日期 | 背景 | 状态 |
|------|------|------|------|
| ... | ... | ... | ... |

## 🔮 下周计划
- 基于本周进展，下周需要关注：...
- 未完成的任务：...

## 🤔 本周反思
- 做得好的：...
- 需要改进的：...
- 新的认知：...
```

### 操作 3: 月报生成

**目标**：聚合本月所有笔记，生成月度总结报告。

**流程**：类似周报，但范围是整月，且增加：
- 月度趋势分析（和上个月对比）
- 重点项目进展汇总
- 决策汇总表格
- 月度反思

### 操作 4: 主题复盘（Topic Review）

**目标**：按标签或关键词聚合相关笔记，生成深度复盘报告。

**流程**：

1. 用户指定主题（如"支付"、"AI 产品"）
2. 在 vault 中搜索所有相关笔记（按标签 `#支付` 或关键词匹配）
3. 按时间排序，梳理：
   - 讨论演进时间线
   - 核心决策及变更
   - 关键洞察汇总
   - 遗留问题
4. 生成复盘报告

**主题复盘输出模板**：

```markdown
---
type: topic-review
topic: 支付系统
date: 2026-05-23
notes_count: 12
date_range: 2026-01-15 ~ 2026-05-20
---

# 主题复盘：支付系统

## 📈 讨论演进时间线

| 时间 | 事件 | 相关笔记 |
|------|------|----------|
| 2026-01-15 | 初始方案讨论 | [[初始方案]] |
| 2026-03-10 | 技术选型确定 | [[技术选型]] |
| 2026-05-01 | 重构启动 | [[重构计划]] |

## 🎯 核心决策汇总

| 决策 | 时间 | 当时理由 | 当前评价 |
|------|------|----------|----------|
| 选择方案 A | 03-15 | 性能最优 | ✅ 验证通过，性能提升 40% |
| 放弃方案 B | 03-15 | 维护成本高 | ✅ 正确，后续证明 |

## 💡 关键洞察
1. **洞察一**：...（来自 [[笔记1]]、[[笔记2]]）
2. **洞察二**：...（来自 [[笔记3]]）

## ❓ 遗留问题
- [ ] XX 问题尚未解决，需要进一步调研
- [ ] YY 方案的长期效果待观察

## 🔗 所有相关笔记
- [[笔记1]]（2026-01-15）
- [[笔记2]]（2026-02-20）
- ...
```

### 操作 5: MOC 内容地图自动生成

**目标**：自动扫描 vault 文件夹结构，为每个文件夹生成 `_MOC.md` 导航页。

**流程**：

1. 扫描 vault 的目录结构
2. 对每个包含 `.md` 文件或子目录的文件夹：
   - 生成或更新 `_MOC.md`
   - 列出该文件夹下所有笔记的 `[[wikilink]]`
   - 列出子文件夹的链接
   - 如果笔记有 frontmatter 中的 `tags`，按标签分组
3. 在父文件夹的 MOC 中添加指向子 MOC 的链接

**MOC 输出示例**：

```markdown
---
type: moc
generated: 2026-05-23
---

# 📂 支付项目 — 内容地图

## 子目录
- [[架构设计/_MOC]] — 架构设计相关文档
- [[会议记录/_MOC]] — 会议记录归档

## 核心文档
- [[支付系统PRD]] — 产品需求文档
- [[技术选型报告]] — 方案对比与决策

## 按标签
### #prd
- [[支付系统PRD]]
- [[支付流程优化方案]]

### #meeting
- [[2026-05-20 支付评审会议]]
- [[2026-05-15 技术方案讨论]]

### #decision
- [[技术选型决策]]
- [[上线方案决策]]
```

## 脚本

### scripts/aggregate-notes.py

扫描和聚合 Obsidian 笔记的辅助脚本。

```bash
python3 scripts/aggregate-notes.py <vault路径> [选项]

选项：
  --date-from <日期>     起始日期 (YYYY-MM-DD)
  --date-to <日期>       结束日期 (YYYY-MM-DD)
  --tag <标签>           按标签筛选 (如 "#支付")
  --type <类型>          按笔记类型筛选: article, meeting, review, all
  --keyword <关键词>     全文搜索关键词
  --output-format <格式> 输出格式: summary, json, markdown
  --top-dir <目录>       只搜索指定子目录
```

### scripts/generate-mocs.py

自动生成 MOC 内容地图的脚本。

```bash
python3 scripts/generate-mocs.py <vault路径>

功能：
  - 扫描所有文件夹
  - 为每个文件夹生成 _MOC.md
  - 自动建立笔记间的层级关系
  - 支持 --dry-run 预览不写入
```

## 陷阱和注意事项

1. **标题文件名规范化**：笔记标题可能包含文件名非法字符（`/`, `\`, `:`, `*`, `?`, `"`, `<`, `>`, `|`）。生成文件名时替换为 `-`。
2. **编码问题**：中文内容始终使用 UTF-8 编码读写。
3. **大 vault 性能**：笔记很多时，按日期范围或标签限定搜索范围，不要全量扫描。
4. **Wikilink 准确性**：只在笔记确实存在时才创建 `[[wikilink]]`。用 `search_files` 验证。
5. **深化不覆盖**：每次深化追加新章节，保留历史深化记录，可以看到思考演进。
6. **幂等性**：周报/月报如果已存在，重新生成时覆盖整篇（因为是基于最新数据的重新汇总）。
7. **MOC 不要过度生成**：只给包含 3 篇以上笔记的文件夹生成 MOC，避免噪音。
